## 1. 项目概述

### 1.1 项目背景

本项目是一个面向艺人谢可寅（Shaking Chloe）的**独立展示型网站**，核心目标是为粉丝和公众提供一个集中、美观、信息完整的谢可寅个人资料门户。网站以"安利"为核心定位，通过高质量的视觉呈现和结构化的内容组织，全面展示谢可寅的演艺生涯、音乐作品、影视作品、舞台活动及成长轨迹。

网站由粉丝发起，采用**工程化、商业化标准**进行设计与开发，非Demo性质，追求可长期维护、可扩展的架构。

### 1.2 目标用户

| 用户群体 | 特征 | 核心需求 |
|---------|------|---------|
| **核心粉丝（虎卫队）** | 深度关注谢可寅动态，希望获取完整历史资料 | 时间线完整的成长史、高质量图库、便捷的分类检索 |
| **路人/潜在粉丝** | 通过作品或热搜了解谢可寅，希望快速建立认知 | 直观的视觉印象、代表作品一键浏览、作品跳转观看 |
| **行业人士** | 媒体、品牌方、合作方 | 专业的作品履历、获奖信息、联系方式 |

### 1.3 核心定位

- **类型**: 内容展示型门户网站（非社交、非电商）
- **访问模式**: 公开访问，无需登录注册
- **内容策略**: 以作品为核心，时间线为骨架，视觉为灵魂
- **技术特色**: 集成AI智能问答（基于MOI知识库），提供谢可寅相关问题的智能解答

## 4. UI/UX 设计需求

### 4.1 整体风格定位

基于参考样本分析，网站采用**沉浸式娱乐媒体风格**：

- **深色模式 (Dark Mode)** 为主基调：深灰/纯黑背景，营造影院级沉浸感，突出彩色内容图片
- **高级感点缀色**：金色/亮黄色作为强调色（呼应VIP、重点推荐、激活状态）
- **现代简洁**：无衬线字体（如苹方/思源黑体），大量留白，信息密度高但不杂乱

### 4.2 布局结构

| 区域 | 设计需求 |
|------|---------|
| **导航** | 左侧固定侧边栏（图标+文字）+ 顶部搜索+用户区，经典"L型"布局 |
| **首屏** | 全宽Hero Banner，高质量人物海报，底部轮播指示器，右侧快速跳转列表 |
| **内容区** | 模块化区块，每个区块有明确的标题+装饰线，内部采用卡片网格 |
| **作品墙** | 规整的响应式网格，卡片圆角处理，悬停显示详情遮罩 |
| **时间线** | 纵向中轴线设计，节点左右交替排列，节点间有连线 |
| **相册** | Masonry瀑布流或等宽网格，支持模式切换 |
| **问答** | 右下角悬浮球 → 展开为聊天窗口，类现代IM界面 |

### 4.3 配色方案

```
主背景色:    #0a0a0a ~ #121212 (深黑/深灰)
次背景色:    #1a1a1a ~ #1e1e1e (卡片背景)
主文字色:    #ffffff (纯白)
次文字色:    #a0a0a0 ~ #b0b0b0 (浅灰，辅助信息)
强调色:      #f5c518 ~ #ffd700 (金色/亮黄，用于激活态、VIP标识、装饰线)
标签色:      半透明渐变色（青绿、蓝色等，用于作品类型标签）
边框/分割线: #2a2a2a ~ #333333
错误色:      #ff4d4f
成功色:      #52c41a
```

### 4.4 字体规范

| 层级 | 字号 | 字重 | 颜色 | 用途 |
|------|------|------|------|------|
| H1 | 32-40px | Bold | 纯白 | 页面主标题、Hero区大标题 |
| H2 | 24-28px | Bold | 纯白 | 区块标题，左侧带金色装饰线 |
| H3 | 18-20px | Semibold | 纯白 | 卡片标题、作品名 |
| Body | 14-16px | Regular | 浅灰 | 正文、描述 |
| Caption | 12-13px | Regular | 深灰 | 日期、标签、辅助信息 |

### 4.5 交互设计

| 交互元素 | 设计规范 |
|---------|---------|
| 卡片悬停 | 轻微放大(scale 1.02-1.05)，显示渐变遮罩+"查看详情"文字 |
| 轮播图 | 自动播放5s间隔，底部圆点指示器，左右箭头控制，支持手势滑动 |
| 横向滚动列表 | 模块右侧显示左右箭头，支持鼠标拖拽/触摸滑动 |
| 分类Tab | 药丸形状或下划线样式，当前选中态高亮 |
| 搜索框 | 胶囊形圆角，聚焦时边框金色高亮，支持快捷键Ctrl+K唤起 |
| 灯箱浏览 | 黑色半透明背景，图片居中，左右箭头+关闭按钮，键盘ESC/左右键支持 |
| 加载状态 | 骨架屏(Skeleton)优先于Spinner，图片区域渐进式模糊加载 |
| 页面过渡 | 路由切换使用淡入淡出，内容区块滚动进入视口时触发淡入动画 |

### 4.6 响应式设计

| 断点 | 布局调整 |
|------|---------|
| >= 1440px | 完整侧边栏+顶部栏，多列网格（4-5列） |
| 1024-1439px | 侧边栏可收起为图标模式，网格3-4列 |
| 768-1023px | 侧边栏隐藏为汉堡菜单，网格2-3列 |
| < 768px | 单列布局，底部固定导航栏，全宽卡片 |

**开发优先级**: 先完成桌面端（>=1024px），再适配移动端。

## 5. 技术架构需求

### 5.1 技术栈选型

| 层级 | 技术选型 | 选型理由 |
|------|---------|---------|
| **前端框架** | React 18 + TypeScript | 组件化、类型安全、生态成熟 |
| **构建工具** | Vite | 快速冷启动、按需编译、现代ESM支持 |
| **样式方案** | Tailwind CSS + CSS Modules | 原子化CSS高效开发，局部样式隔离 |
| **UI组件库** | 自研组件为主，Ant Design/Headless UI为辅 | 高度定制化需求，避免样式冲突 |
| **状态管理** | Zustand / React Query | 轻量级，适合以服务端状态为主的展示型网站 |
| **路由** | React Router v6 | 声明式路由，支持懒加载 |
| **动画** | Framer Motion | 声明式动画，与React深度集成 |
| **图标** | Lucide React | 轻量、现代、树摇优化 |

### 5.2 后端与数据层

| 层级 | 技术选型 | 选型理由 |
|------|---------|---------|
| **数据库** | MatrixOne (HTAP) | 云原生超融合数据库，支持结构化+非结构化数据，MySQL协议兼容 |
| **数据类型** | 结构化（作品/时间线/图片元数据）+ 非结构化（图片文件/Blob） | 统一存储，简化架构 |
| **BaaS/Serverless** | 待定（阿里云函数计算/自建Node服务） | 根据实际访问量评估 |
| **文件存储** | 阿里云OSS | 国内访问速度快，CDN集成 |

### 5.5 架构设计原则

> **高内聚低耦合，低扇入高扇出**

1. **模块化分层**: 前端按页面/功能模块拆分，每个模块独立目录、独立路由、独立状态
2. **数据层抽象**: 数据库访问通过Repository模式封装，上层业务不直接依赖具体数据库实现
3. **接口契约**: 前后端通过OpenAPI/Swagger定义接口，类型自动生成
4. **配置化**: 作品分类、跳转链接、Banner内容等通过数据库/配置文件维护，无需改代码
5. **渐进增强**: 核心内容（作品列表、时间线）支持SSG静态生成，确保SEO和首屏性能；动态功能（问答、搜索）通过CSR增强

## 6. 数据模型需求

### 6.1 核心实体

```
Work (作品)
├── id: UUID
├── title: String (作品名)
├── type: Enum [综艺, 电视剧, 电影, 音乐, 舞台]
├── category: String (细分分类，如晚会/音乐节/单曲/EP)
├── release_date: Date
├── cover_image: URL
├── description: Text
├── role: String (谢可寅角色/身份)
├── platform: String (播出平台)
├── external_link: URL (跳转链接)
├── status: Enum [已播出, 即将上线, 热播中]
├── tags: Array[String]
├── created_at, updated_at: Timestamp

TimelineEvent (时间线事件)
├── id: UUID
├── event_date: Date
├── title: String
├── description: Text
├── category: Enum [选秀, 出道, 影视, 音乐, 获奖, 其他]
├── related_works: Array[Work.id]
├── images: Array[URL]
├── importance: Integer (1-5，控制展示大小)

Photo (图片)
├── id: UUID
├── url: URL
├── thumbnail_url: URL
├── album: String (相册分类)
├── title: String
├── description: Text
├── tags: Array[String]
├── taken_date: Date
├── source: String (图片来源/摄影师)
├── created_at: Timestamp

QA_Knowledge (问答知识)
├── id: UUID
├── question: String
├── answer: Text
├── category: String
├── source: String
├── confidence: Float
├── vector_embedding: Vector (用于语义检索)
```

### 6.2 数据库选型理由

MatrixOne作为HTAP数据库，可同时满足：
- **TP场景**: 作品/图片/时间线的CRUD操作，低延迟事务
- **AP场景**: 后台管理的数据分析、统计报表
- **向量检索**: QA知识库的语义搜索（内置向量检索+BM25）
- **非结构化存储**: 图片Blob、JSON元数据统一存储
